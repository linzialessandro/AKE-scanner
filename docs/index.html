AKE Scanner · interactive lab

See how a sentence behaves as p grows

For many first-order sentences φ, the Ax–Kochen–Ershov principle links truth in the p-adics ℚp (for all large primes p) with truth in the Laurent series field 𝔽p((t)). This lab does not prove theorems about ℚp. It runs constructive checks of φ in 𝔽p((t)) for each prime in a range, then classifies the asymptotic pattern: exceptional primes, a candidate threshold N, and whether the large-p tail is stable.

Same engine as the CLI package ake-scanner (Python via Pyodide). First load fetches the runtime; after that, scans run entirely in your browser — no server-side execution.

How to use this lab

  1. Start with a guided story below — three clicks that show the patterns the scanner is built for (eventually true, always false, mixed).
  2. Watch the prime strip: green = predicate returned true, brick = false, amber = runtime error. For mixed patterns, open the residue analysis and try the suggested modulus.
  3. Read the pattern story, then optionally open CLI text for the same report as ake-scan, or export JSON/CSV. Use Copy link to share a reproducible run.
  4. Explore further: pick another catalog sentence, raise the prime limit to stress-test a threshold, or open the custom playground and write your own predicate(F).

Three shapes to learn first

One click runs a scan with sensible defaults. Look at the strip first, then the pattern badge.

Run a scan

Choose a catalog sentence (or custom code), set the prime range and series precision, then run. Flags mirror the CLI (-s -l -p -v).

Custom predicate playground

Define predicate(F) -> bool (or any predicate_* function). You get F.prime, F.precision, F.t, F.constant(n), and the library Hensel helpers. Code runs only in this browser tab — nothing is uploaded.

Start from a template, tick “use custom code”, then Run scan.

Large limits and ∀-quantifier sentences (over 𝔽p) can take noticeable time. Prefer a Web Worker runtime when available; hard cap is 1000. For mixed patterns, raising the limit often makes the modulus clearer than for eventually-true demos.

    How to read the output
    
      
      
        
    • 
          Pass / fail means the constructive predicate returned
          true or false in 𝔽p((t)) at the chosen truncation precision —
          not a decision procedure for ℚp.
        
      • 
        
    • 
          eventually_true (and dual eventually_false)
          is the AKE-shaped signal: after a finite exceptional set, the scanned
          tail is uniformly true (or false). The tool requires a long enough
          clean tail so one lucky last prime cannot fake stability.
        
      • 
        
    • 
          always_true / always_false means constant on the whole
          scanned range (often a structural obstruction or a trivial witness).
        
      • 
        
    • 
          mixed means no single threshold N — truth usually
          follows a congruence (e.g. Legendre symbol, cyclotomic condition). Use
          residue-class analysis rather than raising the limit alone.
        
      • 
        
    • 
          Errors (amber) are runtime exceptions, kept separate from
          mathematical failures (“no witness” / lift failed).
        
      • 
        
    • 
          False can mean “impossible” or “solver did not find a lift
          at this precision.” Raise precision or inspect the predicate when in doubt.
